# Формирование и интерактивная проработка инструкций

Этот скилл регламентирует начальный этап жизненного цикла инструкций: от сырой идеи оператора до структурированного файла инструкций.

## 📥 Входные данные
- Первоначальное описание требований, концепт или черновик оператора.
- Правила стиля из главного [README.md](../../README.md).

## ⚙️ Алгоритм формирования

1. **Создание структуры файла:**
   - Каждая новая инструкция или набор инструкций оформляется в соответствии с соглашениями об именовании из [README.md](../../README.md) (например, `docs/imp_instructions.md` или `auth.instructions.md`).
   - Файл должен начинаться с раздела `# Goal` (цель) и разделяться на конкретные `Implementation Instructions` (шаги/правила выполнения).

2. **Применение стиля "Лаконичность и полнота":**
   - Выделяйте только **целевые** требования. Избегайте добавления общепринятых, но не упомянутых оператором условий (например, "добавить логгер во все функции", если оператор этого не просил).
   - Оформляйте требования в виде функций: входные данные -> преобразование -> выходные данные.

3. **Интерактивное проектирование (без лишних итераций):**
   - Если требования имеют пробелы (underspecified requirements), агент должен использовать разметку `[!AI-QUESTION]` или `[!AI-ASSUMPTION]` прямо внутри формируемого файла в разделе `Open Questions`.
   - Оператор может ответить на них, обновив файл, что сохраняет контекст и экономит токены.

## 📤 Выходные данные
- Готовый, структурированный файл инструкции с относительной ссылкой на [README.md](../../README.md) для навигации.
