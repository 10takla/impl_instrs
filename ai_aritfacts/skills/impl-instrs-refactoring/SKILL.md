# Рефакторинг и одновременное обновление

Этот скилл управляет процессом модификации существующих инструкций и синхронизации их с кодовой базой.

## 📥 Входные данные
- Запрос оператора на изменение функционала.
- Существующие инструкции и соответствующий им код.

## ⚙️ Алгоритм обновления и рефакторинга

1. **Синхронное изменение (Double-write):**
   - При изменении логики работы системы агент обязан **одновременно** обновить соответствующий файл инструкций и файл реализации (кода). Не допускается ситуация, когда код обновлен, а инструкции устарели.

2. **Рефакторинг структуры инструкций:**
   - Если файл инструкции разрастается, агент должен разбить его на более мелкие файлы (рефакторинг инструкций), сохраняя семантику и расставляя перекрестные ссылки для связности (например, ссылка на родительскую инструкцию или [README.md](../../README.md)).

3. **Инкрементальное обновление реализации (Migration Path):**
   - При получении измененной инструкции агент должен сопоставить её с текущим состоянием реализации и выполнить изменения инкрементально, учитывая уже сделанную работу и сохраняя обратную совместимость.

## 📤 Выходные данные
- Обновленные файлы инструкций.
- Обновленный и протестированный код, строго соответствующий новой версии инструкций.
