# Обратная связь, дебаг и реверс-инжиниринг

Этот скилл отвечает за анализ результатов, обоснование поведения агента и восстановление инструкций по готовому коду.

## 📥 Входные данные
- Код существующей реализации (без инструкций или с частичными инструкциями).
- Ошибки выполнения или запросы оператора на дебаг.

## ⚙️ Алгоритм дебага и обратной связи

1. **Реверс-инжиниринг (Код -> Инструкции):**
   - Если у оператора есть готовый рабочий код без инструкций, агент должен уметь восстановить требования и сформировать для него компактный файл инструкций по правилам "Лаконичности и полноты".

2. **Обоснование результатов выполнения (Post-mortem):**
   - Если агент выполнил инструкцию неверно или возникла ошибка, агент должен сопоставить логи выполнения с текстом инструкции и объяснить оператору, в каком месте возникла неоднозначность или сбой.

3. **Интерактивный опрос (Feedback Loop):**
   - Для уточнения спорных моментов агент формирует точечные вопросы оператору, помогающие улучшить инструкции.

## 📤 Выходные данные
- Восстановленный/скорректированный файл инструкций.
- Описание причин ошибок и рекомендации по доработке промптов.
