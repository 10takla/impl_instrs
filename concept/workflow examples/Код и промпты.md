Этот workflow описывает организацию репозитория таким образом, чтобы совместить работу *инструкций* и *артефактов* с файлами проекта, ручную работку *оператора* с генерацией *агентом*.


**Стандартный общая структура репоизтория:**
- [Файлы инструкций](</concept/Состав пака/Работа с файловой структурой и рабочим пространством/main.md#обнаружение-инструкций>)
- Файлы проекта – Свободно существует в репозитории на любом уровне. Никогда не создаются в инструкциях.
- Инструкции могут создаваться в файлах проекта. Например: 
    ```markdown`
    parser
    |---script.js
    |---`ai_instrs/*`
    ````
- артефакты файлов – `_build/` – обычно в `.gitignore`
- артефакты агента – `ai_artifacts/`

### Организация инструкций

**Инструкции можно описать двумя способами:**
- именованные:
`**/*.ai_instrs/**/*`, `**/ai_instrs.*/**/*`, `**/*.ai_instrs.*`
- неименованные:
`**/ai_instrs/**/*`, `**/ai_instrs.*"`

При неименованных семантически имя передается от родительской директории.


#### Виды организаций инструкций

В любой директории можно орагнизовать инстуркции следующими способами:

1. Файлы инструкций. Например: 
    ```
    parser
    |---Что делает парсер.ai_instrs.md
    |---Зачем нужен парсер.ai_instrs.md
    |---ai_instrs.md (Как главный, но можно без него)
    ```
2. Директории инструкций. Например: 
    ```
    parser
    |---ai_instrs/
    |    |----Действия/
    |    |    |----Что делает парсер.md
    |    |    |----Зачем нужен парсер.md
    |    |----Как обращаться.md
    |    |----main.md – (главный файл директории)
    ```

Для однозначности в директории необходимо иметь только один из этих вариантов. Потому что, например, `ai_instrs.md` пересекается с `ai_instrs/main.md` и т.д.
