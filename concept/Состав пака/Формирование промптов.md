Пак включает инструкции для создания промпта. Результат фиксируется в файловой системе в определённом формате: структура директорий, имена файлов, формат содержимого.

Агент должен учитывать, что оператор итеративно изменяет инструкции напрямую в файловой системе — учитывая это, он минимизирует затраты токенов и операции на обновление контекста.

### Плейсхолдеры (Placeholders)

Инструмент двусторонней мета-коммуникации между оператором и агентом. 
Маркеры служат каналом мета-информации и предназначены строго для формирования инструкций. Их содержимое обязано напрямую касаться логики или текста самой инструкции (например: запрос недостающего контекста для написания или уточнение причин выбранных формулировок). Любой диалог, не обращенный к инструкции, запрещен.

Виды маркеров:

* `{{...}}` — **Оператор → Агент (Делегирование)**. Непосредственное обращение оператора к агенту. Агент выполняет инструкцию внутри скобок и раскрывает плейсхолдер результатом в контексте места его расположения и окружения.
* `[!...]` — **Агент → Оператор (Обратная связь)**. Сигнальный маркер для передачи сообщений, статусов или запросов уточнения от агента. В Markdown и текстовых файлах маркер используется напрямую, в остальных случаях — оборачивается в нативный комментарий целевого файла. Например:
    - Markdown и текст (`md`, `txt`): `[!...]`
    - C-подобные (`js`, `ts`, `rs`, `go`, `cpp`): `// [!...]`
    - Скриптовые и конфиги (`py`, `bash`, `yaml`): `# [!...]`
    - Разметка (`html`, `xml`): `<!-- [!...] -->`
    - Стили (`css`): `/* [!...] */`

    **Примеры тегов:**
    Теги подбираются агентом свободно под конкретную задачу. Возможные варианты:
    - `[!AI-QUESTION]` — агенту не хватает контекста для формирования инструкции.
    - `[!AI-WARNING]` — информации достаточно, но обнаружены потенциальные риски.
    - `[!AI-INFO]` — справочная мета-информация или пояснение логики.

    **Примеры оформления:**
    Жесткого формата (например, обязательного дублирования `//` на каждой строке) не существует. Агент оформляет маркер любым удобным способом, органично вписывая его в нативный синтаксис файла (строчные/блочные комментарии или напрямую в текст):

    *Прямой текст (Markdown / txt):*
    `[!AI-QUESTION] Почему выбран именно этот подход?`

    *Строчные комментарии (Python / Bash):*
    ```python
    # [!AI-WARNING]
    # Формулировка может быть неверно истолкована.
    # Рекомендуется уточнить требования.
    ```

    *Блочные комментарии (JS / CSS):*
    ```javascript
    /* 
       [!AI-INFO] 
       Инструкция успешно адаптирована.
    */
    ```
