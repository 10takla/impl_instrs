# Правила Агента для рабочего пространства (Workspace Agent Rules)

Данные правила переопределяют поведение агента на уровне всего проекта.

## Общие правила для тестов (Test Suite Instructions)

Все тесты, расположенные в поддиректориях `tests/`, должны выполняться по следующим правилам:

1. **Логическая песочница:** Изоляция осуществляется исключительно на уровне области видимости в файловой системе. Агент обязан ограничить все свои операции чтения, поиска и записи рамками папки конкретного выполняемого теста.
2. **Алгоритм тестирования:** Перед запуском любого теста агент обязан ознакомиться с правилами и шагами выполнения, описанными в [tests/AGENTS.md](<tests/AGENTS.md>).

**Если задача не связана с директорией `tests/` — данный раздел не применяется и не влияет на поведение агента.**
