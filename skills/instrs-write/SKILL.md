# Формирование промптов (Prompt Formulation)

Используйте этот skill для создания начальных файлов инструкций и промптов, а также для их базовой инициализации.

## Когда Использовать
* **Явный вызов оператором**: Когда пользователь запрашивает создание новой инструкции или инициализацию промптов.
* **Реагирует на файлы инструкций**: При работе с файлами, соответствующими шаблонам инструкций (`ai_instrs` и т.д.).

## Когда Не Использовать
* Для повседневных или мелких изменений в коде без необходимости создания формальной структуры инструкций.

## Основные Правила

### 1. Соблюдение стиля
* Следуйте правилу [instruction-style.md](../../rules/instruction-style.md).
* Применяйте принцип **целевой (субъективной) полноты** и **лаконичности**. Не пишите лишнего бойлерплейта.

### 2. Функциональная структура
При создании файла инструкции разделяйте его на три логические части:
1. **Входные данные (Inputs)**: Описание начального состояния, файлов или контекста.
2. **Алгоритм (Transformation)**: Пошаговые действия для выполнения.
3. **Выходные данные (Outputs)**: Ожидаемые артефакты.

### 3. Оптимизация и графовая навигация
* Каждая созданная markdown-инструкция должна содержать относительную ссылку на родительский файл инструкции или на главный [README.md](../../README.md).
* Это обеспечивает быструю навигацию по ссылкам без сканирования файловой структуры.
