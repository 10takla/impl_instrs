# Реверс-инжиниринг инструкций (Reverse Engineering Instructions)

Используйте этот skill для восстановления, генерации или декомпозиции инструкций на основе уже существующего кода, файловой структуры или готовых результатов.

## Когда Использовать
* **Явный вызов оператором**: Когда пользователь запрашивает генерацию или восстановление промпта по существующему коду ("сделай инструкцию по этому коду", "опиши как воссоздать этот результат" и т.д.).

## Когда Не Использовать
* При создании инструкций с нуля без готовой реализации.
* При обычном выполнении уже существующих инструкций.

## Основные Правила

### 1. Анализ реализации
* Проанализируйте готовую реализацию (файлы кода, структуру папок, форматы конфигураций).
* Выделите ключевые паттерны, архитектурные решения и используемые зависимости.

### 2. Формулирование инструкций
* Сформулируйте пошаговый алгоритм (Transformation) и входной контекст (Inputs), которые привели бы к созданию данной реализации.
* Опишите выходные артефакты (Outputs) в соответствии со стилем [impl-instrs-instruction-style.md](../../rules/impl-instrs-instruction-style.md).
* Убедитесь, что инструкции не содержат избыточного кода, а описывают логику его создания.
