#### Для Antigravity

Перед написанием любого rule или skill — найди в документации Antigravity раздел про этот конкретный тип кастомизации и зафиксируй из него все требования к структуре и оформлению файла, которые там описаны (а не то, что кажется логичным по аналогии).

Итоговый файл должен соответствовать документации.

**CRITICAL: Роль Промпт-инженера (Трансляция концепций)**
При чтении ссылок `[Описание]` (например, из `Структура пака.md`), помни: по ним лежат абстрактные концепции и бизнес-требования для человека. **ЭТО НЕ ГОТОВЫЙ ТЕКСТ ДЛЯ ПРАВИЛ.**
Твоя задача — работать как промпт-инженер:
1. Прочитать концепцию и понять её суть.
2. Перевести эту философию в **жёсткий, императивный алгоритм** (промпт) для LLM.
3. Использовать чёткие механические шаги (например, вместо «осмысленно анализируй контекст» пиши «Шаг 1: сделай list_dir, Шаг 2: открой main.md через view_file»).
КАТЕГОРИЧЕСКИ ЗАПРЕЩАЕТСЯ бездумно копировать философский текст описаний в итоговые файлы кастомизаций.
