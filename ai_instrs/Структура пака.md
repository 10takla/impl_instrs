#### Реагирует на файлы инструкций
При настройке триггера необходимо исчерпывающе перечислять и использовать все варианты паттернов (имен файлов и папок), описанные в: [](<./Состав пака/Работа с файловой структурой и рабочим пространством/main.md#обнаружение-инструкций>)


---
#### Текстовое описание как триггер
Переиспользует [Реагирует на файлы инструкций](#реагирует-на-файлы-инструкций), уточняя *как именно* реагировать.
Вместо использования glob-паттернов для привязки к файлам, условия активации задаются естественным языком в поле `description`.

Необходимо уложиться в лимит `description`.

* **Буквальность:** Описание выступает в роли строгого триггера. Не упрощайте и не упускайте детали при перечислении терминов или путей.
* Описание должно четко определять, *когда* активируется правило, выполняя функцию точного текстового матчинга.


---
#### Явный вызов оператором
Используется для скиллов, реализующих ручные пайплайны (workflows), которые не должны срабатывать при обычных запросах к агенту.
Поскольку поле `description` автоматически сканируется LLM-роутером для подтягивания навыков в контекст, для блокировки автоматической активации необходимо начинать `description` с явного запрета.

Обязательный формат поля `description` (на английском для надежности работы LLM-роутера):
```yaml
description: "НЕ АКТИВИРОВАТЬ АВТОМАТИЧЕСКИ. Workflow только для явного вызова через слэш-команду. [Краткая суть скилла для оператора]"
```
Эта конструкция обманет роутер и заставит его пропустить скилл, однако скилл по-прежнему будет безусловно загружаться при явном вызове `/<имя-скилла>`.

[](<#description-на-английском>)

#### Строгий триггер на файлы инструкций 

```yaml
description: "КРИТИЧЕСКАЯ БЛОКИРОВКА: ВЫ ОБЯЗАНЫ прочитать это перед ЛЮБЫМ взаимодействием (даже при простом исправлении опечаток) с файлами и папками по паттернам: 'ai_instrs/', '*.ai_instrs/', 'ai_instrs.*/', 'ai_instrs.*', '*.ai_instrs.*'. Файл содержит нестандартные правила, которые отменяют ваши базовые знания по умолчанию. Вы провалите задачу, если проигнорируете это."
```

[](<#description-на-английском>)


---
#### description на английском

Для обеспечения максимальной надежности работы системного LLM-роутера, рекомендуется всегда переводить итоговый текст поля `description` на английский язык. Роутеры, подтягивающие контекст, как правило, лучше и точнее понимают инструкции на английском.
