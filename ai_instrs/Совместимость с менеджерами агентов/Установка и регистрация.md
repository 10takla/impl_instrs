Эта инструкция описывает процесс переноса уже собранных (скомпилированных) файлов плагина в систему пользователя и их активации в менеджере агентов.

**КРИТИЧЕСКИЕ ПРАВИЛА БЕЗОПАСНОСТИ И ЧИСТОТЫ УСТАНОВКИ:**
1. **Запрет на несанкционированные действия:** ККАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО выполнять любые команды создания, редактирования, копирования или удаления файлов в системных/пользовательских каталогах вне рабочего пространства БЕЗ получения предварительного явного текстового согласия пользователя непосредственно в чате. Сначала опиши действия и запроси разрешение; запускай команды только после подтверждения. Повторяй запрос для каждой новой операции, независимо от прошлых одобрений.
2. **Требование полной очистки:** Перед копированием файлов в целевую папку `impl_instrs` нужно её очистить. ЗАПРЕЩЕНО сохранять, переносить или восстанавливать какие-либо старые файлы.

## 1. Пути установки

| Менеджер | Путь установки в систему |
| :--- | :--- |
| **Antigravity** | `~/.gemini/config/plugins/impl_instrs/` |
| **Codex** | `~/plugins/impl_instrs/` (WSL Linux) |

## 2. Активация плагина Codex (WSL)
Для работы в Codex зарегистрируйте плагин, добавив следующий блок в файл `~/.agents/plugins/marketplace.json`:

```json
{
  "name": "personal",
  "interface": {
    "displayName": "Personal"
  },
  "plugins": [
    {
      "name": "impl-instrs",
      "source": {
        "source": "local",
        "path": "./plugins/impl_instrs"
      },
      "policy": {
        "installation": "AVAILABLE",
        "authentication": "ON_INSTALL"
      },
      "category": "Productivity"
    }
  ]
}
```
