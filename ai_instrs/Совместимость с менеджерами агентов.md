### Совместимость плагина с Antigravity и Codex

При создании или обновлении кастомизаций обеспечь поддержку работы плагина в обоих агент-менеджерах, соблюдая следующие требования:

- **Имя папки плагина:** Корневая папка плагина в обоих менеджерах должна называться строго `impl_instrs`.

#### 1. Структура манифестов
- **Для Antigravity:** Сохраняй файл `plugin.json` в корневой директории плагина.
- **Для Codex:** Создай директорию `.codex-plugin/` в корне плагина и помести в неё файл `plugin.json`. Пропиши в нём метаданные и путь к скиллам (`"skills": "./skills/"`).

#### 2. Скиллы (Skills)
- Размещай скиллы в директории `skills/{skill-name}/SKILL.md`.
- Указывай в заголовке `SKILL.md` обязательные YAML-поля `name` и `description`. Формат должен быть совместим с обоими менеджерами.

#### 3. Правила (Rules)
- **Для Antigravity:** Размещай файлы правил авто-вызова контекста в директории `rules/` в корне плагина.
- **Для Codex:** Поскольку Codex не считывает правила из папки `rules/` на уровне плагина, задокументируй эти правила в файле `AGENTS.md` или `README.md` в корне плагина с инструкцией для пользователя по их ручному копированию в `~/.codex/rules/` или `.agents/`.

### Установка плагина в систему

#### Среда выполнения и регистрация в Codex в режиме WSL
Codex в режиме WSL выполняется в Linux-окружении (где `~` — это `/root` или домашняя директория пользователя).
Для интеграции плагина (устанавливаемого в `~/plugins/impl_instrs` в WSL) зарегистрируй его в файле `~/.agents/plugins/marketplace.json`:

  Пример конфигурации `marketplace.json`:
  ```json
  {
    "name": "personal",
    "interface": {
      "displayName": "Personal"
    },
    "plugins": [
      {
        "name": "impl-instrs",
        "source": {
          "source": "local",
          "path": "./plugins/impl_instrs"
        },
        "policy": {
          "installation": "AVAILABLE",
          "authentication": "ON_INSTALL"
        },
        "category": "Productivity"
      }
    ]
  }
  ```
