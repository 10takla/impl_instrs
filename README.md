# Пак системных промптов для работы с инструкциями (Implementation Instructions Pack)

Этот пак представляет собой набор правил, соглашений и скиллов для агентов (таких как Antigravity, Codex и др.), направленный на то, чтобы максимально быстро и качественно формировать, обновлять и выполнять инструкции (промпты) в рамках рабочей директории с минимальным расходом токенов.

---

## 📖 Термины и определения

* **Оператор** — пользователь (человек), управляющий агентом.
* **Агент** — ИИ-исполнитель инструкций (например, Antigravity, Codex).
* **Промпты (Инструкции)** — решения, зафиксированные в файловой системе и сохраненные в рамках проекта в виде Markdown-документов.
* **Выполнение/реализация промптов** — процесс исполнения инструкций агентом, ведущий к конечному результату.
* **Решения** — действия агента или оператора (как выполненные, так и принятые к исполнению).
* **Проект** — сочетание файловой структуры рабочего пространства с результатами работы агента.
* **Пак** — набор системных промптов (в формате скиллов).

---

## 📂 Работа с файловой структурой и обнаружение инструкций

### Границы адресации
Рабочая директория проекта является **верхним пределом адресации** для агента. 
* В режиме обдумывания (thinking) агенту предоставлена полная свобода рассуждений.
* При генерации любого вывода (кода, документации, путей) агент должен использовать пути **относительно корня рабочей директории**, полностью исключая абсолютные пути хост-системы.

### Обнаружение инструкций
Инструкции храняются в файлах Markdown (`.md`). Агент автоматически обнаруживает их по следующим паттернам:

1. **Папки (Группы инструкций):**
   * `<keyword>/`
   * `<название>.<keyword>/`
   * `<keyword>.<название>/`
   * *Пример:* `impl_instrs/`, `auth.instructions/`, `instrs.database/`
   * Все `.md` файлы внутри таких директорий считаются файлами инструкций. Не-инструкции (вспомогательные файлы) должны отделяться расширением или размещаться вне этих папок.

2. **Файлы (Инструкции):**
   * `<keyword>.md`
   * `<название>.<keyword>.md`
   * *Пример:* `implementation.instructions.md`, `csv-import.instrs.md`

> [!NOTE]
> `<keyword>` может принимать значения: `implementation_instructions`, `implementation_instrs`, `imp_instructions`, `ai_instructions`, `agent_instructions` или их сокращения (например, `instrs`).

## 🏎️ Оптимизация расхода токенов

Для снижения потребления контекста и токенов соблюдаются следующие правила:
1. **Связи вместо поиска:** Каждый файл инструкций должен содержать явную относительную ссылку на главный [README.md](./README.md) (или родительскую инструкцию). Агент перемещается по ссылкам как по карте, вместо дорогостоящего сканирования директорий.
2. **Локальность изменений:** При изменении требований агент должен обновлять только связанный файл инструкций, не переписывая весь контекст без необходимости.

---

## 🎨 Стиль: "Лаконичность и полнота"

Для получения идеального результата в разработке и поддержке промптов агент и оператор следуют принципу целевой детализации:
* **Субъективная (целевая) детальность:** Описывается детально только то, что оператор реально хочет получить, выражая свои мысли и намерения. 
* **Отсутствие бойлерплейта:** Не допускается детальное описание ради "объективной правильности" (например, расписывание стандартных обработок ошибок, если про них не было мыслей у оператора).
* **Паттерн "Функция":** Инструкции структурируются аналогично функциям:
  * *Входные данные:* Что дано агенту на входе.
  * *Тело (Алгоритм):* Что нужно сделать.
  * *Выходные данные:* Ожидаемые артефакты и результаты.
