# Правила и структура тестов (Test Rules & Structure)

Этот файл регламентирует общую структуру организации тестов и строгие правила их выполнения с целью предотвращения утечки ожидаемых результатов в контекст агента. Базовая логика и ограничения описываются здесь, а специфика конкретного теста выносится строго в его `README.md`.

## Общая структура тестов

- **Группировка:** Тесты должны быть организованы иерархически:
  1. Общая директория для проверяемого навыка.
  2. Внутри неё — директория для конкретного правила (например, проверяющая конкретную строку инструкции).
  3. Уже внутри неё — папки самих тестов.
- **Документирование групп:** Каждая директория-группа (как уровня навыка, так и уровня правила) обязана содержать краткий `README.md` (буквально одно предложение) с описанием её назначения.
- **Тест:** Сам тест представляет собой изолированную директорию (название в формате `snake_case`).
- **Состав теста:** Каждый тест обязан содержать как минимум три файла:
  - `README.md` — инструкции по запуску теста.
  - `run.ai_instrs.md` — файл с инструкцией, которую агент должен будет выполнить.
  - `EXPECTED.md` — данные для финальной проверки результата.

## Строгие требования (Порядок выполнения)

Агент обязан соблюдать следующую жесткую последовательность при прохождении любого теста:

1. **Шаг 1: Ознакомление с запуском (Чтение `README.md`)**
   Агент начинает с чтения файла `README.md` в папке теста.
   *Внимание:* Файл обязан быть полностью очищен от ожидаемых результатов, чтобы гарантировать чистоту эксперимента и не влиять на решение агента.

2. **Шаг 2: Изолированное выполнение**
   Агент автономно выполняет задачу на основе полученной инструкции, руководствуясь только глобальными правилами проекта.
   *КАТЕГОРИЧЕСКИ ЗАПРЕЩАЕТСЯ* открывать, искать или читать файл `EXPECTED.md` до полного завершения работы над задачей.

3. **Шаг 3: Проверка результатов (Верификация)**
   Только после того, как задача автономно выполнена и все артефакты сгенерированы, агент переходит к шагу верификации. Он обязан прочитать файл `EXPECTED.md` и сопоставить фактически полученный результат с ожидаемым.
