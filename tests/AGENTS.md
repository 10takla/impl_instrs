# Правила выполнения тестов (Test Execution Rules)

Эти правила регулируют процесс запуска и прохождения тестов в папке `tests/` с целью предотвращения утечки ожидаемых результатов в контекст выполнения.

## Алгоритм выполнения теста:

1. **Шаг 1: Ознакомление с запуском (Чтение README.md)**
   Агент начинает с чтения файла `README.md` в папке конкретного теста. Этот файл должен содержать исключительно инструкции по запуску теста (какую команду выполнить или какой файл инструкций запустить).
   *КАТЕГОРИЧЕСКИ ЗАПРЕЩАЕТСЯ* размещать в `README.md` описание ожидаемых результатов, путей к создаваемым файлам или подсказок по их содержанию.

2. **Шаг 2: Изолированное выполнение**
   Агент выполняет запуск теста на основе полученной инструкции. Во время выполнения агент должен руководствоваться только глобальными правилами проекта, системными промптами и инструкциями самого теста.
   *КАТЕГОРИЧЕСКИ ЗАПРЕЩАЕТСЯ* открывать, искать или читать файл `EXPECTED.md` (или файлы с ожидаемыми результатами) до полного завершения работы над задачей.

3. **Шаг 3: Проверка результатов (Верификация)**
   Только после того, как все файлы/артефакты теста были сгенерированы и сохранены, агент переходит к шагу верификации.
   На этом шаге агент обязан прочитать файл `EXPECTED.md` и сопоставить фактически полученный результат с ожидаемым.
